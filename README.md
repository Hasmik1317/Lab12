#  Լաբորատոր աշխատանք №12-13: Packers, Crackmes և Anti-Debugging

##  Նկարագրություն
Այս լաբորատոր աշխատանքի նպատակն է ուսումնասիրել ծրագրային ապահովման պաշտպանության առաջնակարգ մեթոդները և դրանց շրջանցման (Reverse Engineering) տեխնիկաները Kali Linux միջավայրում։

Աշխատանքը բաղկացած է երեք հիմնական մասից.
1. **Executable Packer** - Ծրագրի բինար կոդի թաքցնում և runtime վերականգնում։
2. **Crackme Challenge** - Գաղտնաբառով պաշտպանված ծրագրի ստեղծում և կոտրում։
3. **Anti-Debugging** - Ծրագրի վերլուծության հայտնաբերում և կանխում։


##  Իրականացված պաշտպանական մեթոդներ

### 1. Simple Executable Packer
Պաշտպանում է ծրագիրը **Static Analysis**-ից (երբ ֆայլը վերլուծվում է առանց գործարկման)։
- **Builder (`packer_builder.cpp`)**: Վերցնում է թիրախային բինար ֆայլը և այն գաղտնագրում է XOR (0xAA) բանալիով։ Արդյունքում ստացվում է `packed.bin` ֆայլը, որը պարունակում է միայն «աղբ» տվյալներ։
- **Stub (`packer_stub.cpp`)**: Սա իրականացնում է **Runtime Unpacking**։ Աշխատելու պահին այն վերծանում է `packed.bin`-ը հիշողության մեջ, ստեղծում է ժամանակավոր ֆայլ, տալիս է կատարման իրավունք (`chmod`) և աշխատեցնում այն։

### 2. Crackme և Reverse Engineering
Սա իմիտացիա է լիցենզիայի կամ գաղտնաբառի ստուգման համակարգի։
- **Crackme (`crackme.cpp`)**: Ծրագիրը պահանջում է գաղտնաբառ, որը պահված չէ բաց տեքստով, այլ օբֆուսկացված է XOR գործողությամբ։
- **Solver (`solve_crackme.cpp`)**: Ցույց է տալիս Reverse Engineering-ի գործընթացը։ Կատարելով հակադարձ մաթեմատիկական հաշվարկ՝ այն վերականգնում է իրական գաղտնաբառը (`HYE@O`)։

### 3. Anti-Debugging (Linux ptrace)
Պաշտպանում է ծրագիրը **Dynamic Analysis**-ից (երբ ծրագիրը վերլուծվում է Debugger-ի տակ)։
- **Մեխանիզմ**: Օգտագործվել է Linux-ի `ptrace` համակարգային կանչը։ Քանի որ մեկ պրոցեսին կարող է կցվել միայն մեկ debugger, ծրագիրը փորձում է `PTRACE_TRACEME` հրամանով կցվել ինքն իրեն։ 
- **Արդյունք**: Եթե ծրագիրը աշխատում է GDB-ի տակ, `ptrace`-ը վերադարձնում է սխալ, ինչը հնարավորություն է տալիս ծրագրին հայտնաբերել դեբագերը և անմիջապես դադարեցնել աշխատանքը։
